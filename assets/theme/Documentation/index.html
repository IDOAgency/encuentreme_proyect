Getting started with your theme is fairly easy. First of all you will need a wordpress install on your server. Once you've got that fresh install up and running, it's time to install our theme.

Installing your theme

There are two ways to install your theme, through the Admin Panel, or via FTP

Admin Panel (Easy)

In your wordpress admin panel; Go to "Appearance > Themes". Once there click "Install Themes". On the new page click "Upload", right under the tabs. Now make sure you upload your zip file for the theme (not what you downloaded from themeforest).

The file you will be uploading is called spotfinder.zip and is located in the folder "Wordpress Files" within your download file.

Once you've finished uploading, click "Activate Theme" (Or altenatively, go to your themes page again and activate SpotFinder).

Upload your download file in this section

Via FTP (Advanced)

Unzip the file spotfinder.zip from your download file. Via FTP, upload the folder you just extracted (spotfinder) to your folder /wp-content/themes/.

Once you've finished installing your theme, let's start with the basics!

The folder you will be uploading is called spotfinder and is located in the folder "Wordpress Files" within your download file.

Dropdown Fields

Watch the narrated tutorial on dropdown fields

Dropdown fields are probably the most used fields in SpotFinder. They work pretty much like categories, but instead of using categories themselves, we have create a set of options that are easier to use.

To create a dropdown field start by selecting "Dropdown" in your field type.

Field Attributes

In the field attributes tab when you select dropdown as your field type we can set a few options:

  • Change map location upon selecting value: Toggle on and off whether the map should change location when the user sets a value for the dropdown. This field is only used whe nyou are using custom values. The location of the map is set in the next tab when we add our values.
  • Dropdown Type: Here you can set whether the values of your dropdown are custom values or just your categories.
  • Dropdown Categories: If you have set your dropdown type as categories, here you can select which categories are actually displayed in the dropdown.

Field Values

If you have set custom values as your dropdown type this is where you set your custom values.

When creating new values you have 4 inputs you can fill out

  • Label: Thisis the label of your value. In other words the title.
  • Latitude: If you have set the Change map location option to true, you can insert here the latitude the map will change to when the user selects this value.
  • Longitude: If you have set the Change map location option to true, you can insert here the longitude the map will change to when the user selects this value.
  • Zoom Level: If you have set the Change map location option to true, you can insert here the zoom level the map should change to when the user makes this selection. Zoom values are between 1 and 20 where 1 is zoomed out and 20 is totally zoomed in.

Front-end Options

This is your front-end options and where we set most of the visual stuff.

  • Enable on Front End: If set to true, this search field value will be shown in the front end on map overlays, results lists and listing page. This is good when your search field is a value that the user needs to know the value for. For example when we are dealing with Property type for example, we can set this to true so the user sees it when he hovers the pin etc.
  • Front End Markup: If you have set the enable on front end to on, this is the markup that is shown. For example, if the user has selected "Townhouse" for our property type, this field is where you set how the value is outputted. You can use the characted "%" to replace it with the value the user has chosen. So for example if the user has chosen townhouse, we could set our markup to "Property Type: %" - So in the front end it would show "Property Type: Townhouse". You can also wrap the main label in span tags to make the label bolder. (example <span>Property Type: </span> % - Would display Property Type: Townhouse. For more information on this field view this
  • Enable on Sort By: If set to true the user will be able to sort his results based on this search field. Only avaialble for numeric field. Not available for dropdowns

Submission Options

This is where we can set a few submission options. Please refer to the Public Submissions settings for a more in-depth look at this

  • Enable Field on Public Submissions: Whetehr or not this field is available for setting during public submissions. If set to off the user will not see this during his submisison
  • Required: If available in our public submissions, make this a required field, forcing the user to seelct at least one value.
  • Categories: Choose what categories this field applies to. This comes down to category speicific fields. Meaning that the user can only select this field, if he has selected on the categories chosen.
  • Description: Description for this field on public submissions - HTML accepted
  • Dropdown Multiple or Single: Whether to let the user select only one value or multiple ones.
  • Search field price: Set this to 0 or leave blank to not charge the user. If set to a numeric value, the user will ahve to pay this value to be able to set this field.
  • Cart description: Description that shows in the cart and checkout when the user adds this field to the cart.

Radius - Advanced Geolocation

Watch narrated video for the 1.2 Update with geolocation

SInce spotfinder 1.20 we have introduced the radius option for dropdown, and dependent dropdown fields. When you have the Change map location upon selecting value turned on, we are able to select a radius for each and every value.

The difference between using and not using a radius is that whe na radius is selected, we ignore the selection of this dropdown fiwld for the listings, and filter based on the radius each field has.

For example. Suppose we have a dropdown field called City. If we are NOT using radius, we will get the user's selection of city, and filter every listing that has that "city" assigned to it. Now if er ARE using radius, we do not take into consideration which city a listing has assigned to it, we take into consideration the center point of the dropdown item (latitude and longitude) and we draw a radius around it. And instead of filtering by the selection of the search field, we include every listing that is within this radius.

Keep in mind that doing queries with radius as such can be quite slow. If you are planning on running a website that has more than 1000 listings I would highly recommend not using radius as this can slow down websites on shared hosts and etc.

To set a radius for a dropdown, simply create a new dropdown, type in your latitude and longitude, and set a radius. Radiuses can be set in either kilometres or miles, depending on what you have set in your custom admin panel under Map Options > Geolocation Options

In order for the radius to work, you must turn on the option Change map location upon selecting value as well as have a valid latitude and longitude for each value.

Dependent Dropdowns

Watch the narrated tutorial on dependent dropdown fields

Dependent dropdowns work just like dropdown fields, but instead they need a parenting dropdown in order to work. Dependent Dropdowns are a very good way to create a chain of selections for your user. The basic concept of Dependent dropdowns is that; Depending on the selection of its parent dropdown, the values of the depend dropdown change, to match the selection of its parent.

Think of it this way, Imagine we are creating two dropdowns - Country and Area. In country we can select which country we want to look at, and based on the selection of my country, I want the user to be able to select the area of that country. Now because Areas vary from country to country, you would like to have different values for different countries, and dependent dropdowns do just that.

When you select your search field as a dependent dropdown, the process of configuring it is pretyt much the same as the dropdown with a few additions. In this section we will only refer to the extra options and differences. For more information on other fields check out the previous section.

Dependent Dropdowns for Categories

From SpotFinder 1.35 you can now use dependent dropdowns with categories. All you have to do is create a parent dropdown with categories (nto custom values), and create a new dependent dropdown that has that dropdown as the parent.

Make sure you only select parent categories when creating your parent dropdown.

Field Attributes

  • Parent Dropdown: This is your parent dropdown. Select one of the dropdowns you have already created. Once you have made this selection, save as a draft or publish it so the theme can load the options from the parent dropdown

Field Values

This works just like the dropdown values with the addition to the parent item field. The parent field is the one of the fields from your parent dropdown.

When you add a new value to a parent, it tell the theme that these are the values that should be shown when the user selects the parent. So for example, if we have East Coast and West Coast under the United States parent item, when the user selects the field United States in his parent dropdown, the dependent search field will automatically populate itself with West Coast and East Coast

For more information on the other fields refer to the Dropdown subsection

Min. & Max. Fields

Watch the narrated tutorial on minimum and maximum fields

Minimum and maximum fields are numeric fields which work as a limiter when it comes to filtering. Whenever in place, maximum or minimum falue fields filter the listings based on the value the user hsa chosen.

A minimum value will let the user choose a numeric value, where the theme will then take that value, and only display lists that have that value or higher. For maximum value fields the other way around; The theme will then take the value, and limit listings that have that specific value equal or lower to the value the user has chosen.

To give you a better example, let's think of it as star rating. For a minimum value, if the user has set to say, 3, the theme will then look for listings that have a minimum rating value of 3. For maximum fields it would only look for listings with the rating of 3 or less.

A good thing about min. and max. fields is that they work like dropdowns, so you can actually limit what the user can choose.

Field Attributes

This field does not take any attributes

Field Values

You will note that this section is very similar to the dropdown section. Here you can insert a label (this works as a title in your dropdown) and a numeric value.

Your numeric value, is a whole number which the theme uses to filter the listings.

Unlike dropdowns, your first value is the default value for your search field. You can also leave the field value blank. It is recommended that you set your first value as blank, this way the theme will ignore this field.

If we were setting a ratings field for example, I would start with a value labeled Any and a value blank. From there I would start with a value labeled 1+ Stars and a value of 1

Range Fields

Watch the narrated tutorial on range and check fields

Range fields is a combination of minimum and maximum fields, enabling your user to set a range of a numeric value that filters your results. The main difference is the way it is displayed in the front end and that you need to set a minimum and a maximum value.

Instead of creating a set of pre defined values, you set a minimum and a maximum value as well as an increment step. The increment step defines how the user slides it, 1 by 1, 5 by 5 and so forth.

Field Attributes

  • Minimum Value: The minimum value the user can set in the range field.
  • Maximum Value: The maximum value the user can set in te range field
  • Increment Steps: When the user uses the slide, specify the steps in how the slide increments. So instead of when he slides it increases/decreases 1 by 1, it can be 5 by 5, 1000 by 1000 and so forth
  • Label: The label that is displayed in the front end. This is the displayed chosen range of the user. Here the sign % is replaced with the value. So for prices for example you could set $% which in the front end it would display something like $10 - $150
  • Enable Input: If togled off, the user is forced to use one of the values of the slider when submitting, otherwise he is able to insert a custom value that is not a part of one of the increments.

For all the other fields please refer to the other sections - dropdown section

Check Fields

Watch the narrated tutorial on range and check fields

Check fields are basic true of false fields and do not require any sort of options aprt from front-end submissions and front end options which we cover in the dropdown section.

Front-End Markup

When creating your search fields you can select whether or not the values chosen are displayed in the front-end.

If enable you can choose the way the value is displayed. You have two options, you can set a static text or use the string "%%" to replace it with the value the user has chosen.

For example, Say we have a search field called "Country" where the user is able to select from a number of different contries. For the purpose of this example let's say the user has chosen "Brazil". We can either just use %% where this piece of string would be replaced with the value, in our xase in the front end it would display as Brazil

We can also insert labels, so instead of just displaying the name Brazil, we could use instead Country: %% where this would display Country: Brazil

You can also make the use of <span> tags to further style it. span tags are by default, bold and have a lighter color. IN this case we would the use <span>Country:</span> Brazil, where in the front-end this would display Country: Brazil

Multiplying Labels

This proves helpful when using fields that are numeric so we can have icons. In our Restaurant & BArs Live preview you'll see that we have stars as the rating. For this purpose we use the keyword %_*VAL_% THE_ELEMENT.

By using the keyworkd %_*VAL_% we tell the theme to multiply whatever comes after it. So if we replace the string THE_ELEMENT with the image of a star or with an fontawesome icon, we can achieve this result.

Search Forms

Search forms are the second part of your search functionality. Once you have created your search fields you can assign them to search forms. Search forms are assigned to listing pages in which they impact the result of your listings.

For your listings to work properly, you need at least one search form in your website.

Overview

Search forms are located right under your Search fields in the admin panel, and you can start adding a new one by going to Search Forms > Add New

First thing you'll note when adding a new search form is that the panel looks pretty similar to our search fields. This is an easy to use, drag & drop panel, so no dramas here.

bPanel Options

For your search form we only need to set a few options. In your bPanel, navigate to Search Options > Main Options. Here are the options

  • Search Bar: This is the search form that is displayed in your homepage. If you have more than one please make sure that you select the appropriate form here.
  • Search Bar Position: The position of your search bar in relation to the slider. This can either be positioned at the top or bottom of your slider.
  • Initial Visibility: Whether the search bar should always be visible or only visible when hovering it.
  • Enable User Notifications for Matching Submissions: Whether or not to enable user notifications. Refer to the User Notification section for more information on this.

In Spotfinder you can set unique sidebars and assign them to different pages. This means that you can have different sidebar content in different pages. There are two way of adding custom sidebars.

Creating new sidebars

To create new sidebars you can either jsut select (add new sidebar) when selecting your new sidebar in the Pages panel or you can do that through the bPanel under Content Options > Custom Sidebars

Assigning Sidebars

To assign a custom sidebar to one of your pages or post, simply select it at the bottom of your right sidebar in the dashboard when adding/editing a page or a post

Deleting Sidebars

To delete a custom sidebar, in the bPanel under Content Options > Custom Sidebars you can then select the sidebars you want and hit Remove Sidebars

Shortcodes

Shortcodes in Spotfinder are very use to use and you can find a full list of them in our live previews. However There is a couple of things we need to talk about:

Shortcode Generator

The shortcode generator is an easy to way to easily implement shortcodes into your pages without having to look into the documentation. It is located right under your content area when adding/editing pages and posts.

You can just select the shortcode you want and the generator will walk you the process of filling in the attributes.

Spots Shortcode

There is a shortcode that was built specifically for SpotFinder. It displays a slider or a list of listings and it takes several attributes. You can use the shortcode generator to use this shortcode, Just wanted to point out that it is there.

Widgets

Spotfinder counts with a number of custom widgets that you can use with our theme. Obviously you can use third party widgets but in this section we will cover the widgets that are shipped out with the theme.

Widget Areas

There are several widget areas within spotfinder that you can add in your widgets. Here they are

Sidebars that are not listed here are either custom sidebars and/or footer widget areas that you set in the admin panel

  • General Sidebar: This is the all-site sidebar. Whenever you want to display something that will be displayed in every single sidebar within your theme this is where you will drop widgets
  • Page Sidebar: Displayed in pages when using the default sidebar
  • 404 Sidebar: Displayed in your 404 - Not found page
  • Search Sidebar: Displayed when the user uses a search field for spotfinder
  • Listings Sidebar: Displayed in your listings page (when using a sidebar)
  • Spots Sidebar: Displayed in the sidebar that shows in the single spot page
  • Blog/News Sidebar: Displayed in your blog page
